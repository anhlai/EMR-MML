1. はじめに

　MML Inspectorは、MML (Medical Markup Language) Version 3.0.2で記述された
MMLドキュメントの内容を表示するプログラムです。

　MMLドキュメントのMmlHeaderと各セクションのdocInfoおよびMMLコンテンツ
モジュールの内容を個別に表示します。MML Inspectorの基本レイアウトでは、
下記のMMLコンテンツモジュールに対応しています。

・患者情報モジュール (PatientModule)
・健康保険情報モジュール (HealthInsuranceModule)
・診断履歴情報モジュール (RegisteredDiagnosisModule)
・生活習慣情報モジュール (LifestyleModule)
・基礎的診療情報モジュール (BaseClinicModule)
・初診時特有情報モジュール (FirstClinicModule)
・経過記録情報モジュール (ProgressCourseModule)
・手術記録モジュール (SurgeryModule)
・臨床サマリモジュール (SummaryModule)
・検歴情報モジュール (TestModule)
・報告書情報モジュール (ReportModule)
・紹介状モジュール (ReferralModule)
・予約請求モジュール (ClaimModule)
・点数金額モジュール (ClaimAmountModule)

　DTDによるドキュメントの検証は行わず、可能な限り内容を表示します。基本的に、
MMLタグ毎にMMLタグ名(に準拠)を項目名、MMLタグのテキストを値、属性値を[]で囲んで
列挙して、1行で表示します。

　<項目名>: <値> [ <属性値1>, <属性値2>, ... ]

属性値の表示順は、MML Version 3.0.2規格書の定義における出現順です。また、
MMLタグのテキストにXHTMLタグが使用可能なデータについては、XHTMLタグを無視して
表示します。

　基本レイアウト以外にも、同梱の開発リソースを使用してMMLドキュメントレイアウト
プラグインモジュールを作成することにより、目的に合ったレイアウトでMMLドキュメント
を表示することができます。複数のMMLドキュメントレイアウトプラグインモジュールを
同時に適用することが可能で、それらに実装されたレイアウトは動的に切り替えることが
できます。

　MML Inspectorは、MMLドキュメントの取得方法の違いで、Stand-alone Modeと
Server Modeの2種類のモードで動作します。それぞれのモードは、動的に切り替える
ことができます。

・Stand-alone Mode

　プログラム組み込みのMMLファイル読み込み機能を使用して、ファイル形式の
MMLドキュメントを読み込むことができます。また、同梱の開発リソースを使用して
MMLドキュメント読み込みプラグインモジュールを作成することにより、データベースや
サーバプログラムからなど、さまざまなデータソースからMMLドキュメントを
読み込むことが可能になります。

・Server Mode

　クライアント-サーバモデルのサーバとなり、TCP Socketを使用してクライアント
から1MMLドキュメント/1コネクションでMMLドキュメントを受信します。
1MMLドキュメントを受信するためのアプリケーションレベルの通信手順は、同梱の
開発リソースを使用してMMLドキュメント受信プラグインモジュールとして実装します。


　プログラムの検証は、MedXMLコンソーシアム様ホームページ(www.medxml.net)の
MML Version 3.0.2規格書のサンプルインスタンスで行っています。


2. リリースファイル一覧

　MML Inspectorリリースファイル一式は、以下の通りです。

  MmlInspector\
   |
   +-bin\
   |  +-BasicLayout.dll   … ライブラリファイル
   |  +-MmlInspector.exe  … 実行ファイル
   |  +-MmlLayout.dll     … ライブラリファイル
   |  +-MmlLoader.dll     … ライブラリファイル
   |  +-MmlReceiver.dll   … ライブラリファイル
   |
   +-dev\
   |  +-MmlLayout\     … MMLドキュメントレイアウトプラグインモジュール開発リソース
   |  |  +-assembly\             … アセンブリ
   |  |  +-sample\               … サンプル
   |  |  +-MmlLayout_Manual.txt  … 説明書
   |  |
   |  +-MmlLoader\     … MMLドキュメント読み込みプラグインモジュール開発リソース
   |  |  +-assembly\             … アセンブリ
   |  |  +-sample\               … サンプル
   |  |  +-MmlLoader_Manual.txt  … 説明書
   |  |
   |  +-MmlReceiver\   … MMLドキュメント受信プラグインモジュール開発リソース
   |     +-assembly\               … アセンブリ
   |     +-sample\                 … サンプル
   |     +-MmlReceiver_Manual.txt  … 説明書
   |
   +-Readme.txt    … リリースメモ (このファイル)


3. 動作環境

　Microsoft Windows XP (SP3) / Windows Vista (SP1) / Windows 7 / Windows 8
　Microsoft .NET Framework 3.5

　※ Windows 7 では、Microsoft .NET Framework 3.5のインストールの必要はありません。

4. インストール

　a. Microsoft .NET Framework 3.5をインストールしてください。
　　 (Windows 7では.NET Framework 3.5のインストールの必要はありません。)
　b. 任意のフォルダにリリースファイル一式を展開してください。

　レジストリは使用していませんので、モジュールの登録作業などは必要ありません。


5. アンインストール

　リリースファイル一式を削除してください。


6. 使用方法

・Stand-alone Mode

(1) 画面右上の「Open MML」ボタンでMMLドキュメントを読み込みます。
    →MMLドキュメント読み込みプラグインモジュールが存在する場合は、MMLドキュメント
      読み込みプラグインモジュールを使用してMMLドキュメントを読み込みます。
    →MMLドキュメント読み込みプラグインモジュールが存在しない場合は、組み込みの
      MMLファイル読み込み機能を使用します。

・Server Mode

(1) 画面右上の「Port:」に表示されたポート番号でクライアントの接続を待ちます。
    →MMLドキュメント受信プラグインモジュールが存在しない場合、Server Modeは
      動作しません。
    →ポート番号を変更した場合、「Restart」ボタンでServer Modeを再起動して
      ください。

(2) クライアントからMMLドキュメントを受信すると、受信したMMLドキュメントが
    表示されます。
    →「Received Sound」チェックボックスをチェックすると、MMLドキュメント
      受信時にサウンドが鳴ります。


・モード共通

(1) 画面左側に、MMLドキュメントのMMLヘッダとセクションの一覧が表示されます。
(2) 一覧から、いずれかの項目を選択すると、選択した項目の内容が表示されます。
(3) 表示された内容の各要素をマウス左クリックすると、画面下部にその要素を
    記述したMMLタグ(とその配下)を表示します。
(4) MMLヘッダまたはセクション毎に、利用可能なレイアウトがLayoutコンボボックスに
　　列挙されます。利用可能なレイアウトは、MMLドキュメントレイアウトプラグイン
　　モジュールを作成して増やすことができます。


7. 各種プラグインモジュールの作成および使用方法

　MMLドキュメント読み込みプラグインモジュールの作成および使用方法については、
dev\MmlLoaderディレクトリ内のMmlLoader_Manual.txtを参照してください。

　MMLドキュメント受信プラグインモジュールの作成および使用方法については、
dev\MmlReceiverディレクトリ内のMmlReceiver_Manual.txtを参照してください。

　MMLドキュメントレイアウトプラグインモジュールの作成および使用方法については、
dev\MmlLayoutディレクトリ内のMmlLayout_Manual.txtを参照してください。
